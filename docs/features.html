HyperMark Editor (v1.0) #

HyperMark is Oxidian's revolutionary block-based Markdown editor ‚Äî the editor of the third kind. It's not a plain textarea, and it's not WYSIWYG. It's both.

How It Works

  • Unfocused blocks render as real HTML ‚Äî headings, bold, links, code blocks all display rich
  • The focused block (where your cursor is) shows raw Markdown in a full CodeMirror 6 editor
  • Transition is seamless ‚Äî ~100ms animated switch when you move between blocks
  • Source of truth is always pure Markdown files on disk. Always.

Key Capabilities

  • Block-aware editing ‚Äî Drag & drop blocks, slash commands, block menus (Notion-style UX)
  • AI-native ‚Äî Ghost-text completions like Cursor. Inline streaming, Tab to accept
  • Vim-ready ‚Äî First-class Vim mode plugin, not an afterthought
  • Custom block types ‚Äî Create new block renderers in ~20 lines via the plugin API
  • Performance ‚Äî Virtual viewport renders only visible blocks. 60fps at 10,000+ lines

Tech Stack

  • Data structure: Rope Buffer (O(log n) edits)
  • Parsing: tree-sitter-markdown (WASM, incremental)
  • Active block: CodeMirror 6 (instance pooling for performance)
  • Preview: markdown-it ‚Üí HTML
  • Viewport: Custom virtual scroller

Focus Mode (v0.9) #

Focus Mode provides a distraction-free writing environment by hiding all UI elements except the editor content.

What Gets Hidden

  • Sidebar and file tree
  • Ribbon bar
  • Status bar
  • Tab bar (single-note view)

Toggle Focus Mode with Ctrl+Shift+F or via the command palette (Ctrl+P ‚Üí "Toggle Focus Mode"). Press the same shortcut or Esc to exit.

Split Panes #

Work on multiple notes side by side using split panes. This is especially useful for referencing one note while writing another, or comparing two documents.

How to Split

  • Drag a tab to the right side of the editor to create a split view. A drop overlay appears when you drag a tab toward the edge.
  • You can also right-click a file in the sidebar and select "Open in Split Pane".

Working with Panes

  • Each pane has its own tab bar ‚Äî you can have multiple tabs in each pane
  • Resize panes by dragging the divider handle between them
  • Both panes support full editing with auto-save, preview, and all keyboard shortcuts
  • Close the split by closing all tabs in the right pane, or dragging the last tab back to the left

Graph View #

The graph view visualizes the connections between your notes as an interactive force-directed graph. Click the üó∫Ô∏è graph icon in the sidebar ribbon or use the command palette (Ctrl+P ‚Üí "Open graph view").

How It Works

  • Every note appears as a node (circle) in the graph
  • Wiki-links ([[note name]]) create edges between nodes
  • Nodes with more connections appear larger
  • The graph uses a physics simulation ‚Äî connected nodes attract, unconnected nodes repel

Full-Text Search #

Oxidian includes a powerful full-text search engine powered by Tantivy ‚Äî the Rust equivalent of Apache Lucene.

Using Search

  • Press Ctrl+Shift+F to open the search panel in the sidebar
  • Type at least 2 characters to start searching
  • Results show the title, path, and a text snippet with context around the match
  • Click a result to open the note
  • Results are ranked by relevance score

Search Features

  • Title and body search ‚Äî searches both note titles and full content
  • Tag search ‚Äî click a tag in the tags panel to search for all notes with that tag
  • Instant results ‚Äî search triggers 250ms after you stop typing
  • Auto-indexing ‚Äî notes are indexed automatically when saved

The search index is stored in .search_index/ inside your vault. If search results seem stale, you can delete this folder and Oxidian will rebuild the index on next launch.

Encrypted Vaults #

Oxidian can encrypt your entire vault using AES-256-GCM encryption with Argon2id key derivation ‚Äî the same standards used in password managers.

Setting Up Encryption

Enable Encryption

Go to Settings ‚Üí Vault ‚Üí Encryption and toggle it on. Or enable it during the onboarding wizard.

Set a Master Password

Enter and confirm a strong master password. This password derives the encryption key using Argon2id.

Unlock on Launch

When you open an encrypted vault, a password prompt appears. Enter your password to decrypt and access your notes.

How It Works

  • Each note is encrypted individually when saved
  • Encrypted notes are stored as JSON containing: salt (Base64), nonce (Base64), and data (Base64 ciphertext)
  • A unique random salt (32 bytes) and nonce (12 bytes) are generated for each encryption operation
  • The search index stores plaintext content for searching ‚Äî keep this in mind for your threat model

Changing Your Password

Go to Settings ‚Üí Vault ‚Üí Change Password. You'll need to enter your current password and the new password.

‚ö†Ô∏è

Critical: There is no password recovery mechanism. If you forget your password, your encrypted notes cannot be decrypted. Keep a backup of your password in a secure location (e.g., a password manager).

View Modes (v1.1) #

Oxidian now supports three editing modes, matching Obsidian's workflow. Toggle between them with Ctrl+E or using the icons in the top-right toolbar.

  • Live Preview ‚Äî Default mode. Unfocused blocks render as rich HTML; the focused block shows raw Markdown (HyperMark behavior).
  • Source Mode ‚Äî Plain Markdown editing across the entire document. No rendered blocks.
  • Reading Mode ‚Äî Read-only rendered view. No editing ‚Äî just clean, styled output.

View mode is per-tab and persisted across sessions, so each note remembers how you last viewed it.

Auto-Updater (v1.1) #

Oxidian checks for updates automatically on startup by comparing your version against the latest GitHub Release using semver comparison.

  • Update banner appears at the top with changelog summary and an install button
  • Download progress displayed inline with auto-restart on completion
  • Configurable in Settings ‚Üí General:
    • Auto-check for updates ‚Äî toggle startup check on/off
    • Auto-install updates ‚Äî download and install without prompting
‚ÑπÔ∏è

The update system uses reqwest (Rust HTTP client) and semver comparison on the backend. No telemetry is sent ‚Äî it only fetches the GitHub Releases API.
