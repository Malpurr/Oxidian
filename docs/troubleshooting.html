Troubleshooting

Solutions for common issues. If you don't find your problem here, open an issue on GitHub.

App Won't Start #

Missing shared libraries

If you see errors about missing .so files, you're missing system dependencies. Oxidian (via Tauri) requires:

  • WebKitGTK 4.1 ‚Äî libwebkit2gtk-4.1
  • GTK 3 ‚Äî libgtk-3
  • GLib Networking ‚Äî glib-networking (for TLS/SSL)
  • libsoup 3 ‚Äî HTTP client library

Debian/Ubuntu:

bash 
sudo apt install libwebkit2gtk-4.1-dev libgtk-3-dev \
  libayatana-appindicator3-dev librsvg2-dev \
  glib-networking libsoup-3.0-dev

Fedora:

bash 
sudo dnf install webkit2gtk4.1-devel gtk3-devel \
  libsoup3-devel glib-networking

Arch Linux:

bash 
sudo pacman -S webkit2gtk-4.1 gtk3 libsoup3 glib-networking

Permission denied

If using the AppImage, make sure it's executable:

bash 
chmod +x Oxidian-*.AppImage

NixOS: Dynamically Linked Executable #

On NixOS, you'll see this error when trying to run the AppImage or pre-built binary:

text 
error: cannot execute: required file not found
or
error: a]'ELF' interpreter not found

This happens because NixOS doesn't have a standard /lib/ld-linux-x86-64.so.2 dynamic linker. There are several solutions:

Solution 1: nix-shell (Quick)

bash 
nix-shell -p webkitgtk_4_1 gtk3 glib-networking \
  openssl pkg-config libsoup_3 --run ./Oxidian-*.AppImage

Solution 2: buildFHSEnv (Permanent)

Create a file oxidian.nix for a persistent FHS-compatible environment:

nix 
{ pkgs ? import <nixpkgs> {} }:

(pkgs.buildFHSEnv {
  name = "oxidian";
  targetPkgs = pkgs: with pkgs; [
    webkitgtk_4_1
    gtk3
    glib-networking
    openssl
    libsoup_3
  ];
  runScript = "/path/to/Oxidian-x86_64.AppImage";
}).env

Solution 3: Build from Source

Build Oxidian directly with cargo on NixOS ‚Äî this avoids all dynamic linking issues. See the Building from Source guide.

Blank Screen on Launch #

If Oxidian opens but shows a white or blank screen:

  1. Check WebKitGTK version ‚Äî You need WebKitGTK 4.1 (not 4.0). Older versions may cause rendering issues.
  2. GPU driver issues ‚Äî Try disabling hardware acceleration: 
    WEBKIT_DISABLE_COMPOSITING_MODE=1 ./Oxidian-*.AppImage
  3. Wayland issues ‚Äî If on Wayland, try forcing X11: 
    GDK_BACKEND=x11 ./Oxidian-*.AppImage
  4. Check logs ‚Äî Run from terminal to see error output: 
    RUST_LOG=debug ./Oxidian-*.AppImage 2>&1 | tee oxidian.log

Plugin Not Working #

If a plugin doesn't load or behaves unexpectedly:

Step 1: Check the Console

Open the developer tools (usually Ctrl+Shift+I in the Tauri window, or check the terminal output) and look for:

  • Stub warnings: [Oxidian] Stub: ClassName.method not fully implemented ‚Äî The plugin is calling an API that Oxidian doesn't fully support.
  • Error messages: [PluginLoader] Failed to load plugin xyz: ... ‚Äî The plugin threw an error during initialization.
  • Unknown require: [PluginLoader] Plugin xyz tried to require unknown module: ... ‚Äî The plugin needs a Node.js module that isn't available.

Step 2: Verify Plugin Files

  • Ensure manifest.json has a valid id, name, and version
  • Ensure main.js exists and is the compiled JavaScript (not TypeScript source)
  • Check that the plugin folder name matches the id in manifest.json

Step 3: Check Compatibility

Plugins that heavily depend on CodeMirror 6 editor extensions, the Workspace Leaf API, or Canvas features are unlikely to work. See the Known Limitations section.

Notes Not Saving #

  1. Check vault path permissions ‚Äî Ensure Oxidian has write access to your vault directory: 
    ls -la /path/to/your/vault/
  2. Disk space ‚Äî Ensure you have free disk space
  3. File locked ‚Äî Another program may have the file open. Close other editors.
  4. Check console for errors ‚Äî Look for "Failed to save" messages in the terminal

Oxidian auto-saves 2 seconds after your last edit. You can also force a save with Ctrl+S.

Encryption Issues #

Forgot password

üîí

There is no password recovery. Oxidian uses AES-256-GCM with Argon2id key derivation. The password is never stored ‚Äî only used to derive the encryption key. If you forget your password, your encrypted notes cannot be recovered.

Prevention: Store your vault password in a password manager (1Password, Bitwarden, KeePassXC, etc.). Consider keeping an unencrypted backup of critical notes.

"Decryption failed: wrong password or corrupted data"

  • Double-check your password for typos
  • Check Caps Lock
  • If the file was edited outside Oxidian, the encrypted format may have been corrupted

Mixed encrypted/unencrypted notes

When you enable encryption on an existing vault, only newly saved notes are encrypted. Previously saved notes remain as plain markdown until you open and re-save them. Oxidian detects the format automatically ‚Äî encrypted notes start with {"salt":"....

Search Not Working #

Stale results or missing notes

The search index may become out of sync if files were edited outside Oxidian. To rebuild:

  1. Close Oxidian
  2. Delete the .search_index/ folder inside your vault
  3. Restart Oxidian ‚Äî the index will be rebuilt automatically
bash 
rm -rf /path/to/vault/.search_index/

Search returns no results

  • Queries must be at least 2 characters
  • Tag searches (clicking a tag) automatically strip the # prefix
  • Try simpler search terms ‚Äî Tantivy uses a query parser that may interpret special characters

Graph View Empty #

The graph shows connections created by wiki-links. If your graph is empty or shows only disconnected dots:

  • Add wiki-links: Use [[Note Name]] syntax in your notes to create connections
  • Standard markdown links don't count: Only [[wiki-links]] create graph edges, not [text](url) links
  • Refresh: The graph data is fetched from the backend. If you just added links, close and reopen the graph view

Common Error Messages #

ErrorCauseSolution
Failed to create/open index Search index directory is corrupted or locked Delete .search_index/ and restart
Could not find home directory $HOME environment variable not set Set HOME env var: export HOME=/home/user
Key derivation failed Argon2id error (rare, usually system memory issue) Ensure sufficient RAM available; try restarting
Cipher init failed Invalid key length for AES-256 Likely a bug ‚Äî please report it
Failed to create index writer Tantivy can't allocate memory for indexing (50MB default) Ensure sufficient free RAM; close memory-heavy applications

HyperMark / Block Editor Issues #

Block transitions feel laggy

  • Ensure hardware acceleration is enabled in your system
  • Very large blocks (1000+ lines in a single block) may cause slower transitions ‚Äî consider splitting into smaller sections
  • Check if a plugin is injecting heavy DOM modifications

Focused block shows raw Markdown but doesn't switch to rich preview

  • Click outside the block or press ‚Üì/‚Üë to move focus ‚Äî the block should render as HTML
  • If blocks stay raw, try reloading the window (Ctrl+R)

Focus Mode won't activate

  • Ensure you're in the editor view (not graph view or settings)
  • Try the command palette: Ctrl+P ‚Üí "Toggle Focus Mode"
  • Check if a plugin is intercepting Ctrl+Shift+F

Bookmarks not persisting

  • Bookmarks are stored in .oxidian/bookmarks.json inside your vault
  • Ensure the vault directory is writable
  • If the file is corrupted, delete it and restart ‚Äî bookmarks will reset

Outline Panel not showing headings

  • The Outline Panel parses headings from the current note's Markdown. Ensure you're using standard # Heading syntax
  • HTML headings (<h1>) inside Markdown are not detected

Clicking empty space below content doesn't focus the editor

  • Fixed in v1.1.0. Update to the latest version.
  • In older versions, you had to click directly on a text block. Now clicking anywhere in the editor area (including empty space below your content) activates the last block with the cursor at the end.

View mode not switching with Ctrl+E

  • Requires v1.1.0+. Older versions don't have view mode toggling.
  • Check if a plugin is intercepting Ctrl+E
  • Use the view mode icons in the top-right toolbar as an alternative

Auto-updater not detecting new versions

  • Ensure "Auto-check for updates" is enabled in Settings ‚Üí General
  • Oxidian checks the GitHub Releases API ‚Äî ensure you have internet access and GitHub isn't blocked
  • Manual check: visit the Releases page to see if a newer version is available

Getting Help #

If your issue isn't covered here:

  • GitHub Issues: Open an issue with as much detail as possible (OS, steps to reproduce, error messages)
  • Logs: Run Oxidian from the terminal with RUST_LOG=debug and include the output in your report
  • Console: Check the browser developer console for JavaScript errors
