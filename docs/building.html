Building from Source

How to build Oxidian from source, set up a development environment, and contribute to the project.

Prerequisites #

You'll need the following tools installed:

Rust

Oxidian's backend is written in Rust. Install it via rustup:

bash 
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
# Restart your shell, then verify:
rustc --version   # Requires Rust 2021 edition (1.56+)
cargo --version

Node.js

The Tauri CLI and build tools require Node.js:

bash 
# Using nvm (recommended):
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install 22
nvm use 22

# Verify:
node --version   # v22.x recommended
npm --version

Build Steps #

Clone the Repository

git clone https://github.com/nicokimmel/oxidian.git
cd oxidian

Install Node Dependencies

npm install

This installs the Tauri CLI (@tauri-apps/cli) and the Tauri API (@tauri-apps/api).

Build the Release Binary

npm run tauri:build

This compiles the Rust backend in release mode and bundles the application. The output is in src-tauri/target/release/.

The build produces:

  • src-tauri/target/release/oxidian â€” The standalone binary
  • src-tauri/target/release/bundle/deb/ â€” Debian package (.deb)
  • src-tauri/target/release/bundle/appimage/ â€” AppImage
â±ï¸

First build takes time. Cargo needs to compile all Rust dependencies (Tauri, Tantivy, AES-GCM, etc.). Subsequent builds are much faster thanks to incremental compilation.

Development Mode #

For development with hot-reload:

bash 
npm run tauri:dev

This runs cargo tauri dev which:

  • Compiles the Rust backend in debug mode (faster compilation, slower runtime)
  • Opens the app window with live reload â€” changes to frontend files (src/) are reflected immediately
  • Rust changes require a recompile (automatic on save)
  • Developer tools are available via right-click â†’ Inspect or Ctrl+Shift+I

Contributing #

Oxidian welcomes contributions! Here's how to get started:

Getting Started

  1. Fork the repository on GitHub
  2. Clone your fork: git clone https://github.com/YOUR-USERNAME/oxidian.git
  3. Create a branch: git checkout -b feature/my-feature
  4. Make changes and test with npm run tauri:dev
  5. Commit with a clear message: git commit -m "feat: add xyz feature"
  6. Push and open a Pull Request

Good First Issues

Look for issues labeled good first issue â€” these are beginner-friendly tasks.

Code Style

  • Rust: Follow standard rustfmt formatting. Run cargo fmt before committing.
  • JavaScript: Use ES modules, consistent indentation (4 spaces), and JSDoc comments for public functions.
  • Commits: Use conventional commits format: feat:, fix:, docs:, refactor:, etc.
