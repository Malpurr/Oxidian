Getting Started

Welcome to Oxidian! This guide will walk you through installation, setup, and everything you need to start taking notes like a pro.

Installation #

Linux (Pre-built Binary)

Download the latest release from the GitHub Releases page. Oxidian is available as an AppImage and a .deb package.

bash 
# Make the AppImage executable and run
chmod +x Oxidian-*.AppImage
./Oxidian-*.AppImage

NixOS / Nix

On NixOS, dynamically linked binaries won't run directly. Use nix-shell with the required dependencies:

bash 
# Enter a shell with required dependencies
nix-shell -p webkitgtk_4_1 gtk3 glib-networking \
  openssl pkg-config libsoup_3 --run ./Oxidian-*.AppImage

Alternatively, build from source (see Building from Source) or use buildFHSEnv for a permanent solution.

Build from Source

If you prefer to compile Oxidian yourself, see the Building from Source guide for full instructions.

System requirements: Oxidian uses Tauri (WebKit-based) on Linux. You'll need WebKitGTK 4.1, GTK 3, and related libraries. Most desktop Linux distributions have these pre-installed.

First Launch & Onboarding #

When you start Oxidian for the first time, a setup wizard guides you through the initial configuration:

Welcome Screen

Click "Get Started" to begin the setup process.

Set Up Your Vault

Choose between Create New Vault (starts with an empty folder) or Open Existing Vault (use a folder of existing markdown notes). Select a path or browse to a directory. The default location is ~/.oxidian/vault/.

Vault Security

Optionally enable AES-256-GCM encryption with Argon2id key derivation. Set a master password ‚Äî this cannot be recovered if lost. You can skip this and enable encryption later from Settings.

Quick Tour

A brief overview of Oxidian's key features: the markdown editor, graph view, full-text search, daily notes, and slash commands. Click "Start Writing" to enter the app.

Creating and Editing Notes #

There are several ways to create a new note:

  • Press Ctrl+N to open the new note dialog
  • Click the üìÑ New Note button in the sidebar toolbar
  • Click a [[wiki-link]] to a note that doesn't exist yet ‚Äî Oxidian will create it for you
  • Use the slash command / menu for quick formatting

With the HyperMark editor (v1.0+), unfocused blocks render as rich HTML while the focused block shows raw Markdown with full CodeMirror 6 editing. You can also toggle a traditional split-view preview with Ctrl+E.

‚ú®

New in v0.9: The sidebar now includes Bookmarks (‚≠ê pin your favorite notes), Recent Files (üïê recently opened notes), and an Outline Panel (üìë heading navigation). You'll also see Reading Time and word count for each note.

Notes are auto-saved 2 seconds after your last edit. You can also save manually with Ctrl+S.

Using Markdown #

Oxidian uses standard CommonMark markdown (rendered by pulldown-cmark) with extensions for wiki-links, tags, and callouts.

Headings

markdown 
# Heading 1
## Heading 2
### Heading 3
#### Heading 4
##### Heading 5
###### Heading 6

Cycle through heading levels with Ctrl+H.

Text Formatting

markdown 
**Bold text**           ‚Üí Ctrl+B
*Italic text*           ‚Üí Ctrl+I
~~Strikethrough~~       ‚Üí Ctrl+Shift+X
`Inline code`           ‚Üí Ctrl+`

Lists

Numbered lists:

Lists auto-continue when you press Enter. Press Enter on an empty list item to break out of the list.

Wiki-Links #

Wiki-links are the heart of connected note-taking. Link to other notes using double brackets:

markdown 
Check out [[My Other Note]] for more details.
Also see [[projects/oxidian-notes]].

When you click a wiki-link:

  • If the linked note exists, it opens in the editor
  • If it doesn't exist, Oxidian creates a new note with that name and opens it

Wiki-links also power the Graph View and Backlinks ‚Äî they form the connections between your notes. Insert one quickly with the slash command /wikilink or just type [[.

üí°

Tip: Use descriptive note names. Wiki-links use the filename (without .md), so [[Rust Error Handling]] is more useful than [[note17]].

Tags #

Organize your notes with hashtag-style tags:

markdown 
# Meeting Notes

Discussed the #project-roadmap with the team.

#meeting #2026 #important

Tags appear in the Tags panel in the sidebar (click the tag icon in the ribbon). Clicking a tag performs a search for all notes containing that tag.

Daily Notes / Journaling #

Oxidian has built-in support for daily journaling. Press Ctrl+D or click the üìÖ icon in the ribbon to create (or open) today's daily note.

Daily notes are stored in the daily/ folder with the format YYYY-MM-DD.md. Each note is created with a template:

markdown 
# Monday, February 09, 2026

## Journal



## Tasks

- [ ] 

## Notes

Command Palette #

Press Ctrl+P to open the command palette ‚Äî a quick-access launcher for all available commands:

  • Built-in commands: Create new note, open daily note, save, search, graph view, settings, toggle preview, close tab
  • Plugin commands: Any commands registered by loaded plugins appear here too

Type to fuzzy-filter commands, use ‚Üë/‚Üì to navigate, and Enter to execute. Press Esc to close.

‚ö°

Power tip: The command palette is the fastest way to access any feature. If you forget a keyboard shortcut, just press Ctrl+P and type what you want to do.
