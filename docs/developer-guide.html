System Dependencies (Linux)

# Ubuntu/Debian
sudo apt install libwebkit2gtk-4.1-dev build-essential curl wget \
  libssl-dev libgtk-3-dev libayatana-appindicator3-dev librsvg2-dev \
  libsoup-3.0-dev libjavascriptcoregtk-4.1-dev

# Fedora
sudo dnf install webkit2gtk4.1-devel openssl-devel gtk3-devel \
  libsoup3-devel javascriptcoregtk4.1-devel

# Arch
sudo pacman -S webkit2gtk-4.1 base-devel openssl gtk3 libsoup3

Development Setup #

bash
# Clone the repository
git clone https://github.com/nicokimmel/oxidian.git
cd oxidian

# Install frontend dependencies
pnpm install

# Start development mode (hot-reload)
pnpm tauri dev

This starts both the Rust backend and the frontend dev server with hot-reload.

Adding Tauri Commands #

To add a new IPC command:

1. Define the Rust handler

rust
#[tauri::command]
fn my_command(path: String) -> Result<String, String> {
    // Your logic here
    Ok("result".to_string())
}

2. Register in main.rs

rust
tauri::Builder::default()
    .invoke_handler(tauri::generate_handler![
        my_command,
        // ... other commands
    ])

3. Call from frontend

javascript
const result = await invoke('my_command', { path: '/some/path' });

Code Style #

  • Rust: Follow rustfmt defaults. Run cargo fmt before committing.
  • JavaScript: No framework, vanilla JS. Keep it simple and readable.
  • CSS: Catppuccin color palette. Use CSS custom properties from the theme.
  • Commits: Conventional commits (feat:, fix:, docs:, chore:)

Contributing #

  1. Fork the repository
  2. Create a feature branch: git checkout -b feat/my-feature
  3. Make your changes and test locally with pnpm tauri dev
  4. Run cargo fmt and cargo clippy
  5. Commit with a conventional commit message
  6. Open a Pull Request
