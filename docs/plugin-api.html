Plugin API

How Oxidian's Obsidian-compatible plugin system works ‚Äî supported APIs, limitations, and how to write plugins.

Overview #

Oxidian implements a 3,500+ line compatibility shim that emulates the Obsidian Plugin API. This means many existing Obsidian community plugins work in Oxidian without modification.

How It Works #

graph LR A[Obsidian Plugin JS] --> B[Oxidian Shim Layer] B --> C[App API] B --> D[Vault API] B --> E[Workspace API] B --> F[MetadataCache API] C & D & E & F --> G[Tauri IPC ‚Üí Rust Backend]

Plugins are loaded from the .obsidian/plugins/ directory in your vault. Each plugin's main.js is executed in a sandboxed context with the shim providing Obsidian-compatible APIs.

Installing Plugins #

  1. Open Settings ‚Üí Plugins
  2. Use the Plugin Explorer to browse available plugins
  3. Click Install on any plugin
  4. Toggle the plugin to enable it

Alternatively, manually copy a plugin folder into .obsidian/plugins/ in your vault.

Known Limitations #

  • Plugins relying on Electron/Node.js APIs won't work (Oxidian uses Tauri/WebKit)
  • workspace.getLeaf() returns a simplified leaf object
  • Some advanced Obsidian-specific APIs (e.g., ItemView, FileView) are partially implemented
  • Plugins that use require() for Node.js modules are not supported
  • CSS snippets from Obsidian may need minor adjustments
